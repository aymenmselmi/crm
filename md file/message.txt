# Multi-Tenancy with Separate Databases per Org in NestJS + TypeORM

This is the **database-per-tenant** strategy — the strongest isolation model. Here's how to architect it end to end.

---

## The Core Idea

You'll have two tiers:
1. **Central/public DB** — stores org registrations, tenant metadata, connection configs
2. **Tenant DBs** — one per org, provisioned on registration, contains all their business data

---

## 1. Central DB — Tenant Registry

```typescript
// tenant.entity.ts
@Entity('tenants')
export class Tenant {
  @PrimaryGeneratedColumn('uuid')
  id: string;

  @Column({ unique: true }) 
  slug: string; // used as DB name: tenant_acme

  @Column()
  dbName: string;

  @Column({ default: 'localhost' })
  dbHost: string;

  @Column({ default: 5432 })
  dbPort: number;

  @Column({ default: 'active' })
  status: string;

  @CreateDateColumn()
  createdAt: Date;
}
```

## 2. Tenant Connection Manager

This is the heart of the system — it creates and **caches** DataSource instances per tenant.

```typescript
// tenant-connection.service.ts
@Injectable()
export class TenantConnectionService implements OnModuleDestroy {
  private connections = new Map<string, DataSource>();

  constructor(
    @InjectRepository(Tenant)
    private tenantRepo: Repository<Tenant>,
  ) {}

  async getConnection(tenantId: string): Promise<DataSource> {
    if (this.connections.has(tenantId)) {
      return this.connections.get(tenantId);
    }

    const tenant = await this.tenantRepo.findOneOrFail({ where: { id: tenantId } });

    const ds = new DataSource({
      type: 'postgres',
      host: tenant.dbHost,
      port: tenant.dbPort,
      username: process.env.DB_USER,
      password: process.env.DB_PASS,
      database: tenant.dbName,
      entities: [__dirname + '/../**/*.entity{.ts,.js}'],
      synchronize: false, // use migrations in prod
    });

    await ds.initialize();
    this.connections.set(tenantId, ds);
    return ds;
  }

  async onModuleDestroy() {
    for (const ds of this.connections.values()) {
      if (ds.isInitialized) await ds.destroy();
    }
  }
}
```

## 3. Org Registration Flow — Provision the DB

```typescript
// org.service.ts
@Injectable()
export class OrgService {
  constructor(
    private tenantConnectionService: TenantConnectionService,
    @InjectRepository(Tenant)
    private tenantRepo: Repository<Tenant>,
    private dataSource: DataSource, // the central DS, for raw queries
  ) {}

  async register(dto: RegisterOrgDto): Promise<Tenant> {
    const slug = dto.name.toLowerCase().replace(/\s+/g, '_');
    const dbName = `tenant_${slug}`;

    // 1. Create the physical database
    await this.dataSource.query(`CREATE DATABASE "${dbName}"`);

    // 2. Save tenant record in central DB
    const tenant = this.tenantRepo.create({
      slug,
      dbName,
      dbHost: process.env.DB_HOST,
      dbPort: +process.env.DB_PORT,
    });
    await this.tenantRepo.save(tenant);

    // 3. Run migrations on the new tenant DB
    await this.provisionTenantSchema(tenant);

    return tenant;
  }

  private async provisionTenantSchema(tenant: Tenant): Promise<void> {
    const ds = await this.tenantConnectionService.getConnection(tenant.id);

    // Option A — synchronize (dev only)
    await ds.synchronize();

    // Option B — run migrations (production)
    // await ds.runMigrations();
  }
}
```

## 4. Tenant Context — Extract from Request

Use a middleware to resolve which tenant is making the request (via subdomain, header, or JWT claim):

```typescript
// tenant.middleware.ts
@Injectable()
export class TenantMiddleware implements NestMiddleware {
  constructor(private tenantRepo: Repository<Tenant>) {}

  async use(req: Request, res: Response, next: NextFunction) {
    // Strategy 1: subdomain → acme.yourapp.com
    const host = req.hostname; // acme.yourapp.com
    const slug = host.split('.')[0];

    // Strategy 2: custom header
    // const slug = req.headers['x-tenant-id'] as string;

    const tenant = await this.tenantRepo.findOne({ where: { slug } });
    if (!tenant) throw new NotFoundException('Tenant not found');

    req['tenantId'] = tenant.id;
    next();
  }
}
```

Register it in your AppModule:

```typescript
export class AppModule implements NestModule {
  configure(consumer: MiddlewareConsumer) {
    consumer.apply(TenantMiddleware).forRoutes('*');
  }
}
```

## 5. Using Tenant Connections in Services

```typescript
// product.service.ts
@Injectable()
export class ProductService {
  constructor(private tenantConnectionService: TenantConnectionService) {}

  async findAll(tenantId: string): Promise<Product[]> {
    const ds = await this.tenantConnectionService.getConnection(tenantId);
    const repo = ds.getRepository(Product);
    return repo.find();
  }
}
```

Or via a custom decorator to inject tenant-scoped repos:

```typescript
// tenant-repo.decorator.ts
export const InjectTenantRepository = (entity: EntityTarget<any>) =>
  createParamDecorator(async (_, ctx: ExecutionContext) => {
    const req = ctx.switchToHttp().getRequest();
    const ds = await tenantConnectionService.getConnection(req.tenantId);
    return ds.getRepository(entity);
  });
```

## Key Architecture Decisions

| Topic | Recommendation |
|---|---|
| **DB provisioning** | `CREATE DATABASE` via central DS raw query |
| **Schema setup** | Run TypeORM migrations per tenant (not `synchronize`) |
| **Connection pooling** | Cache connections in a `Map`, lazy-init, destroy on shutdown |
| **Tenant resolution** | Subdomain is cleanest; JWT claim works well for APIs |
| **Migrations** | Keep a shared migrations folder; run via `ds.runMigrations()` per tenant on deploy |
| **Cleanup** | Implement `DROP DATABASE` on org deletion + remove from cache |

## Deployment Tip — Running Migrations Across All Tenants

```typescript
async migrateAllTenants(): Promise<void> {
  const tenants = await this.tenantRepo.find({ where: { status: 'active' } });
  for (const tenant of tenants) {
    const ds = await this.getConnection(tenant.id);
    await ds.runMigrations();
    console.log(`✅ Migrated ${tenant.slug}`);
  }
}
```

Call this from a CLI command or a deploy hook — never from an HTTP handler.
